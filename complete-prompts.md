### API 文档生成
```
角色：技术文档专家
背景：为 [API 名称] API 撰写文档
任务：生成清晰、完整、易于理解的 API 文档
要求：
- 包含端点列表（HTTP 方法、路径）
- 每个端点的请求/响应格式
- 认证说明
- 错误码和错误处理
- 使用示例（curl、JavaScript）
- 速率限制说明

输出格式：
# API 名称

## 概述
[简短描述 API 的用途和目标用户]

## 端点

### POST /auth/login
**用途**：用户登录认证

**请求格式**：
```json
{
  "username": "string",
  "password": "string"
}
```

**响应格式**：
```json
{
  "token": "string",
  "expiresIn": 3600
}
```

## 认证

所有请求需在 Header 中包含：
```
Authorization: Bearer {token}
```

## 错误码

| 错误码 | 说明 | 解决方案 |
|---------|------|----------|
| 401 | 无效的 token | 重新登录 |
| 429 | 请求过多 | 等待并重试 |
| 500 | 服务器错误 | 联系支持 |

## 使用示例

### cURL
```bash
curl -X POST https://api.example.com/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"user@example.com","password":"secret"}'
```

### JavaScript
```javascript
const response = await fetch('https://api.example.com/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username, password })
});

const { token } = await response.json();
```

## 速率限制

- 免费用户：100 请求/分钟
- Pro 用户：1000 请求/分钟
- Enterprise：无限制
```
